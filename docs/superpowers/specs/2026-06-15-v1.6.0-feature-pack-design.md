# clova2Mail v1.6.0 — Feature Pack Design

**Date:** 2026-06-15
**Version target:** 1.6.0
**Status:** Approved (design)

## Overview

Six independent feature epics added on top of the current extract → summarize → send flow. No architectural changes; all epics extend the existing data model, message schema, and UI. Data stays in `chrome.storage.local` (no sync migration this version).

Epics:

- **A. CC/BCC recipients** — per-send selection in the modal
- **B. Multiple summary templates** — named templates, pick at send time
- **C. Model selection** — curated dropdown in popup settings
- **D. Unified editable preview** — raw mode goes through the same editable preview as summary
- **E. Send history** — last 50 sends with full body, view / re-send / delete
- **F. CSV recipient import** — bulk import with dedup + bad-row report

## Cross-cutting decisions

- Storage: stays `chrome.storage.local`. No `storage.sync` migration (deferred).
- Version bump `1.5.3 → 1.6.0` in `manifest.json` and `package.json`.
- Each epic ships with unit + integration tests. Existing pre-existing test failures in the openai/summarize suites are out of scope.
- All new user-facing strings in Korean (matches existing UI).

## Epic A — CC/BCC recipients

**Scope:** Ad-hoc per-send selection only. No persisted CC/BCC defaults on groups (deferred).

### Data / contract
- `src/shared/messages.ts` — `SendEmailRequest.payload` gains `cc?: readonly string[]` and `bcc?: readonly string[]`.
- No `StorageSchema` change.

### Background
- `src/background/index.ts` (SEND_EMAIL handler) — apply the same CRLF sanitization and `EMAIL_REGEX` validation to `cc` and `bcc` as `to`. Empty/absent arrays are valid.
- `src/background/gmail.ts`:
  - `createMimeMessage(to, cc, bcc, subject, htmlBody)` — emit `Cc:` and `Bcc:` headers when non-empty. Gmail's `messages.send` with a raw MIME honors the `Bcc:` header and hides it from recipients.
  - `sendViaGmail(to, cc, bcc, subject, htmlBody)` — thread the new args through.

### UI
- `src/content/modal/RecipientSelector.tsx` — add collapsible **CC** and **BCC** pickers below To, reusing the existing recipient/group source. Default collapsed/empty.

### Error handling
- Invalid cc/bcc address → same error envelope as To (`잘못된 이메일 주소: ...`).

### Tests
- `createMimeMessage` emits Cc/Bcc headers; omits them when empty.
- SEND_EMAIL validates + sanitizes cc/bcc.

## Epic B — Multiple summary templates

**Scope:** Summary templates only. The raw-transcript templates (`DEFAULT_RAW_SUBJECT_TEMPLATE`, `DEFAULT_RAW_BODY_TEMPLATE`) stay single.

### Data / contract
- `src/shared/types.ts` — `EmailTemplate` gains `id: string` and `name: string`.
- `StorageSchema` — replace `emailTemplate: EmailTemplate` with `emailTemplates: EmailTemplate[]` and `activeTemplateId: string`.

### Migration
- `src/shared/storage.ts` — on read, if legacy single `emailTemplate` exists and `emailTemplates` does not, seed `emailTemplates = [{ id, name: '기본', ...legacy }]`, set `activeTemplateId`, and persist. Idempotent.
- New accessors: `getEmailTemplates`, `setEmailTemplates`, `getActiveTemplateId`, `setActiveTemplateId`.

### UI
- `src/popup/components/TemplateEditor.tsx` — template list with add / rename / duplicate / delete (guard: cannot delete the last template; deleting the active one falls back to the first). Edit the selected template.

### Send flow
- Modal shows a template dropdown before summarize/preview; the chosen template drives summarization + variable substitution. Default = `activeTemplateId`.

### Error handling
- Deleting last template blocked with a message. Active template missing → fall back to first.

### Tests
- Template CRUD + last-template guard.
- Legacy `emailTemplate` → `emailTemplates` migration (idempotent).
- Selected template is used for substitution.

## Epic C — Model selection (curated dropdown)

**Scope:** Model id only. Temperature / max_tokens / chunk thresholds stay hardcoded (YAGNI).

### Data / contract
- `src/shared/constants.ts` — `AVAILABLE_MODELS: { id: string; label: string }[]` = `gpt-4o-mini`, `gpt-4o`, `gpt-4-turbo`. Keep `OPENAI_MODEL` as the default id.
- `StorageSchema` — add `model: string` (default `gpt-4o-mini`).
- Token thresholds unchanged (all three models are 128k context).

### Logic
- `src/background/openai.ts` — read the model from storage instead of the `OPENAI_MODEL` constant. If the stored value is not in `AVAILABLE_MODELS`, fall back to the default.

### UI
- Popup settings (extend `AuthSection.tsx` or a small new settings block) — model dropdown bound to storage.

### Tests
- Stored model is sent in the OpenAI request body.
- Unknown/empty stored model falls back to default.

## Epic D — Unified editable preview for raw mode

**Scope:** Raw mode reaches the same editable `PREVIEW` state as summary mode before send.

### Behavior
- `src/content/modal/Modal.tsx` — raw mode produces a variable-substituted subject + body (via `formatRawEmail.ts`) and lands in the editable `PREVIEW` state. Both summary and raw then converge on `PREVIEW → SENDING → SENT`.
- `src/content/modal/formatRawEmail.ts` — ensure its output (filled subject + body) feeds the editable preview.

### Error handling
- No new error paths; reuses existing PREVIEW/SENDING/ERROR states.

### Tests
- Raw mode transitions into editable PREVIEW (not direct send).
- Edits made in raw preview are what gets sent.

## Epic E — Send history (full body, 50-entry ring buffer)

**Scope:** Store full body. Last 50 entries. `storage.local`.

### Data / contract
- `src/shared/types.ts` — `SentEmail = { id: string; sentAt: number; to: string[]; cc: string[]; bcc: string[]; subject: string; bodyHtml: string; mode: 'summarize' | 'raw'; success: boolean; error?: string }`.
- `StorageSchema` — add `sendHistory: SentEmail[]`.

### Storage
- `src/shared/storage.ts` — `appendSendHistory(entry)`:
  - Prepend newest; trim to **50** entries (ring buffer).
  - **Quota guard:** cap a single entry's `bodyHtml` at **256KB** (truncate + mark truncated). If total serialized history still approaches the local quota, drop oldest entries until it fits.
- `getSendHistory()`, `clearSendHistory()`, `deleteHistoryEntry(id)`.

### Background
- `src/background/index.ts` (SEND_EMAIL) — append a history entry on **both** success and failure (record `error` on failure).

### UI
- Popup history section — list (time / subject / recipients / success badge); click to view body; **re-send** (loads recipients + body into the send modal); delete one / clear all.

### Error handling
- Corrupt/oversized entries are truncated, never block sending. History write failure must not fail the send (best-effort, logged).

### Tests
- Ring buffer trims at 50.
- Body over 256KB is truncated + flagged.
- Failure sends are recorded with `error`.

## Epic F — CSV recipient import (dedup + bad-row report)

**Scope:** Bulk import recipients. Dedup by email. Report bad rows; import the good ones.

### Parser
- `src/popup/lib/csvImport.ts` (new, pure function, unit-testable) — parse CSV with header `name,email` (case-insensitive). Handle quoted fields, surrounding whitespace, trailing newline, optional `name`.
- Returns `{ added: Recipient[]; skipped: { row: number; reason: string }[] }`.

### Validation / dedup
- Each row validated with `EMAIL_REGEX`. Invalid rows → `skipped` with reason.
- Dedup against existing recipients **and** within the batch (by lowercased email).

### UI
- `src/popup/components/RecipientList.tsx` — "CSV 가져오기" button → `<input type="file" accept=".csv">` → run import → show summary "N건 추가, M건 스킵" with the skipped-reason list. Persist the `added` recipients.

### Error handling
- Empty file / no valid rows → clear message, no mutation.
- Malformed lines never abort the whole import (per chosen policy: skip + report).

### Tests
- Parser: quoted fields, missing name, blank lines, CRLF/LF.
- Dedup vs existing + within-batch.
- Bad-row reporting; empty-file handling.

## Out of scope (deferred)

- `storage.sync` / Google Drive account-bound storage
- Persisted CC/BCC defaults on groups
- Multiple raw-transcript templates
- Temperature / token-limit tuning UI
- Scheduled send, attachments, i18n, automatic retry/backoff
