# Epic D — Manual Verification Checklist (Unified Editable Preview for Raw Mode)

> **Note (deviation from plan):** The plan assumed no React component-test harness
> existed and made the whole transition manual. Epic F added `@testing-library/react`,
> `@vitejs/plugin-react`, and a `.tsx` include glob, so **Case 1 below is now automated**
> in `tests/unit/modalRawPreview.test.tsx` (it renders the real `<Modal>`, clicks
> "원문 그대로 발송", and asserts the editable PREVIEW with the substituted subject +
> transcript body and that no send fires). The cases below remain manual because jsdom
> cannot exercise real Gmail delivery or a real network failure.

Run after building the extension (`npm run build`) and loading `dist/` as an unpacked
Chrome extension. Open a ClovaNote page with a transcript so the modal opens in
`RAW_DATA_PREVIEW`.

## Case 1 — Raw mode lands in the editable PREVIEW (not a direct send) — AUTOMATED

Covered by `tests/unit/modalRawPreview.test.tsx`. Manual re-check optional:

1. In the modal (`RAW_DATA_PREVIEW`), click **"원문 그대로 발송"**.
2. EXPECT: the modal switches to the PREVIEW view showing a RecipientSelector, a subject
   line reading `[회의록 원문] <회의명> - <오늘 날짜>`, and the rendered transcript body.
3. EXPECT: footer shows **취소 / 수정 / 이메일 발송**, NOT an immediate "발송 중" spinner.
4. PASS if no email is sent before you click **이메일 발송**.

## Case 2 — Edits in the raw preview are what gets sent

1. From Case 1's PREVIEW, click **수정**.
2. Edit the subject input to `EDITED SUBJECT` and edit the body text to include
   `EDITED BODY MARKER`. Click **확인**.
3. Select at least one recipient.
4. Click **이메일 발송**.
5. EXPECT: state goes PREVIEW → SENDING ("이메일을 발송하고 있습니다...") → SENT
   ("이메일이 성공적으로 발송되었습니다!").
6. Check the received email.
   - PASS if the subject is `EDITED SUBJECT` and the body contains `EDITED BODY MARKER`
     (i.e., the edits, not the original formatter output, were sent).

## Case 3 — Summary mode still converges on the same PREVIEW

1. Reopen the modal; click **"AI 요약 후 발송"**.
2. EXPECT: LOADING → PREVIEW with the same editable subject + body UI as raw mode.
3. PASS if summary and raw both reach an identical-looking editable PREVIEW and both
   send via PREVIEW → SENDING → SENT.

## Case 4 — Raw retry path

1. Trigger a send failure (e.g., disconnect network) from a raw-mode PREVIEW.
2. EXPECT: ERROR state with a **재시도** button.
3. Click **재시도**.
4. PASS if the modal returns to the raw editable PREVIEW (not summary, not a direct send).
