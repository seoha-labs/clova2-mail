# Epic C — Model Selection (Curated Dropdown) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let users pick the OpenAI summarization model from a curated dropdown in popup settings, persisted in `chrome.storage.local`, and used by the background summarize path (with a safe fallback to the default when the stored value is unknown).

**Architecture:** Add an `AVAILABLE_MODELS` allow-list constant (keeping `OPENAI_MODEL` as the default id). Add a `model: string` field to `StorageSchema` with `getModel`/`setModel` accessors that default to `OPENAI_MODEL`. In `src/background/openai.ts`, `callOpenAI` receives the resolved model id as a parameter; `summarizeTranscript` reads the stored model once, validates it against `AVAILABLE_MODELS`, falls back to `OPENAI_MODEL` if unknown/empty, and threads it through every API call (single, chunk, merge). The popup adds a `<select>` in `AuthSection.tsx` bound to storage via the existing `useStorage` hook.

**Tech Stack:** TypeScript, React 18, Vitest, `chrome.storage.local`, Tailwind CSS (utility classes only).

Then append to the end of the file (after `setEmailTemplate`):

```typescript
export function resolveModel(candidate: string | undefined): string {
  const known = AVAILABLE_MODELS.some((m) => m.id === candidate);
  return known ? (candidate as string) : OPENAI_MODEL;
}

export async function getModel(): Promise<string> {
  return resolveModel(await get('model'));
}

export async function setModel(model: string): Promise<void> {
  await set('model', resolveModel(model));
}
```

Note: `setModel` also resolves, so an invalid value never reaches storage.
