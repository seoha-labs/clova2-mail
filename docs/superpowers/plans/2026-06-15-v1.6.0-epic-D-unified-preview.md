# Epic D — Unified Editable Preview for Raw Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make raw ("원문 그대로 발송") mode flow through the same editable `PREVIEW` state as summary mode, so a variable-substituted subject + body land in the editable preview and the user's edits there are exactly what gets sent.

**Architecture:** `formatRawTranscriptEmail` (pure) produces `{ subject, htmlBody }`. `Modal.tsx` feeds that output into the existing `subject` / `htmlBody` React state and transitions to `PREVIEW`, where both summary and raw share one editable view. Both converge on `PREVIEW → SENDING → SENT`. The pure formatter and a new pure helper are unit-tested with vitest; the `Modal.tsx` state-flow change is verified via a documented manual checklist because the repo has no React component test harness (see Task 4 rationale).

**Tech Stack:** TypeScript, React 19, vitest (jsdom env), DOMPurify (via `sanitizer.ts`), Chrome extension APIs.

---

## Background: current state of the code (read before starting)

`src/content/modal/Modal.tsx` already routes raw mode through `PREVIEW`:

- `handleRawSend` (lines ~91-98) calls `formatRawTranscriptEmail(...)`, sets `subject` + `htmlBody`, and does `setState('PREVIEW')`.
- The `PREVIEW` render block (lines ~280-322) shows ONE editable subject input + one `contentEditable` body for BOTH modes.
- `handleSend` (lines ~145-178) sends the current `subject` + `htmlBody` state, so edits in the preview already win.

So the convergence largely exists. Epic D's job is to **lock that behavior down** so a future refactor cannot regress it, and to close two real gaps:

1. **Sanitizer inconsistency.** Summary HTML is run through `renderSafeHtml` in `Modal.tsx` before going into the editable preview (`requestSummary`, line ~117). Raw HTML is sanitized *inside* `formatRawTranscriptEmail` via `renderSafeHtml` (markdown → safe HTML) but the Modal does not re-assert it is safe. This is fine today, but the contract "whatever feeds the editable preview is sanitized" is implicit. We make it explicit and tested.

2. **No regression guard.** There is no test asserting raw mode lands in `PREVIEW` (not a direct send) or that the formatter output is what populates the editable fields. A trivial future change to `handleRawSend` (e.g. calling `handleSend` directly) would silently break Epic D with no failing test.

We address both with pure, unit-testable extraction plus a manual checklist for the React state transition.

import type { SendMode } from '../../shared/types';
import { formatRawTranscriptEmail } from './formatRawEmail';

/**
 * The exact shape consumed by the editable PREVIEW state in Modal.tsx.
 * Both raw and summary modes converge on a value of this shape before
 * the user edits and sends.
 */
export interface PreviewModel {
  readonly subject: string;
  readonly htmlBody: string;
  readonly mode: SendMode;
}

/**
 * Builds the preview model for raw mode by forwarding the pure
 * formatter output verbatim. This is the single source of truth that
 * populates the editable subject + body in the modal.
 */
export function buildRawPreview(
  transcript: string,
  meetingTitle: string,
  attendees: readonly string[],
): PreviewModel {
  const { subject, htmlBody } = formatRawTranscriptEmail(transcript, meetingTitle, attendees);
  return { subject, htmlBody, mode: 'raw' };
}
```

import { DEFAULT_RAW_SUBJECT_TEMPLATE, DEFAULT_RAW_BODY_TEMPLATE } from '../../shared/constants';
import { renderSafeHtml, sanitizeHtml, stripHtmlFromTemplate } from '../sanitizer';

export function formatRawTranscriptEmail(
  transcript: string,
  meetingTitle: string,
  attendees: readonly string[],
): { readonly subject: string; readonly htmlBody: string } {
  const today = new Date();
  const date = `${today.getFullYear()}-${String(today.getMonth() + 1).padStart(2, '0')}-${String(today.getDate()).padStart(2, '0')}`;

  const safeTitle = stripHtmlFromTemplate(meetingTitle).replace(/[\r\n]/g, ' ');
  const subject = DEFAULT_RAW_SUBJECT_TEMPLATE
    .replace('{title}', safeTitle)
    .replace('{date}', date);

  const attendeesStr = attendees.length > 0 ? attendees.join(', ') : '-';

  const bodyMarkdown = DEFAULT_RAW_BODY_TEMPLATE
    .replace('{title}', meetingTitle)
    .replace('{attendees}', attendeesStr)
    .replace('{date}', date)
    .replace('{transcript}', transcript);

  const htmlBody = sanitizeHtml(renderSafeHtml(bodyMarkdown));

  return { subject, htmlBody };
}
```
